# Statsez API — LLM Reference

> Bilingual (English / Português). [EN] = English. [PT] = Português.

---

## Overview / Visão Geral

[EN] Statsez API provides **historical football data** from 750+ worldwide leagues.
All data is post-match — no livescores.

[PT] A Statsez API fornece **dados históricos de futebol** de 750+ ligas.
Todos os dados são pós-partida — sem livescore.

---

## Base Configuration / Configuração

- **Base URL:** `https://api.statsez.com/v1`
- **Auth:** Header `x-api-key: YOUR_KEY`
- **Format:** JSON (UTF-8)
- **Rate Limit:** 100 req/min per IP
- **Quota:** Bi-weekly cycle (15 days), resets automatically
- **Quota Headers:** `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`

## Endpoints

### 1. LIST LEAGUES
`GET /football/leagues`
- Params: `country` (optional — filter by country name, e.g. "Brazil")
- Returns all competitions from 750+ leagues across 170+ countries

```bash
curl -H "x-api-key: YOUR_KEY" "https://api.statsez.com/v1/football/leagues?country=England"
```
```json
{

  "data": {
```

`GET /football/fixtures/{matchId}`
- Returns full match data: teams (with logos), scores (FT + HT), tournament, venue, referee
- Also fetches match stats automatically (xG, possession, shots, etc.)
- **This is the primary endpoint for match data**

### 3. STANDINGS
`GET /football/standings`
- Params: `league` (**required** — tournament_stage_id), `season` (optional)
- Returns full league table: position, team, W/D/L, goals, points
- **Important:** The `league` param here is the `tournament_stage_id`, not the league slug. You can find this ID from fixture details (`tournament.tournament_stage_id`).

```bash
curl -H "x-api-key: YOUR_KEY" "https://api.statsez.com/v1/football/standings?league=dINOZk9Q"
```
```json
{
  "success": true,
  "data": {
    "league": "LaLiga",
    "country": "Spain",
    "season": "",
    "standings": [
      {
        "position": 1,

        "goalsAgainst": 39,
        "goalDifference": 63,
        "points": 88
```

### 7. FIXTURES BY LEAGUE
`GET /football/fixtures`
- Params: `league` (**required**), `season`, `team`, `round`, `dateFrom`, `dateTo` (all optional)
- **Note:** This endpoint currently returns empty results. Use fixture details (endpoint 2) with a known match ID instead.

### 8. TEAMS
`GET /football/teams`
- Params: `league` (**required**), `season`, `search` (all optional)
- **Note:** This endpoint currently returns empty results. Team data is available via fixture details and standings.

---

### 9. TEAM FIXTURES
`GET /football/teams/{teamName}/fixtures`
- Params: `league` (**required**), `season` (optional)
- **Note:** This endpoint currently returns empty results.

### 10. LEAGUE SEASONS
`GET /football/leagues/{leagueId}/seasons`
- Returns available seasons for a league
- **Note:** Currently returns limited data (0-1 seasons).

### 11. LEAGUE STATS
`GET /football/leagues/{leagueId}/stats`
- Params: `season` (optional)

## Error Codes

| Code | Meaning |
|------|---------|
| 400 | Invalid parameters or unsupported sport |
| 401 | Missing or invalid API Key |
| 403 | Subscription inactive, expired, or wrong sport |
| 404 | Resource not found |
| 429 | Quota exhausted or rate limit exceeded |
| 502 | Upstream data source error |

## Quick Start (Python)

```python
import requests

KEY = "se_live_your_key_here"
BASE = "https://api.statsez.com/v1"
H = {"x-api-key": KEY}

# List leagues in England
leagues = requests.get(f"{BASE}/football/leagues?country=England", headers=H).json()

# Get match details + stats (primary way to get match data)
match = requests.get(f"{BASE}/football/fixtures/ngWM9fgd", headers=H).json()
print(match["data"]["fixture"]["homeTeam"]["name"])  # "Leeds"
print(match["data"]["stats"]["Ball possession"])      # ["38%", "62%"]

# Get lineups
lineups = requests.get(f"{BASE}/football/fixtures/ngWM9fgd/lineups", headers=H).json()
print(lineups["data"]["home"]["formation"])  # "4-2-3-1"

# Get standings (requires tournament_stage_id)
standings = requests.get(f"{BASE}/football/standings?league=dINOZk9Q", headers=H).json()
print(standings["data"]["standings"][0]["team"])  # "Barcelona"
```

## Quick Start (JavaScript)

```javascript
const KEY = "se_live_your_key_here";
const BASE = "https://api.statsez.com/v1";
const headers = { "x-api-key": KEY };

// Get match with stats
const match = await fetch(`${BASE}/football/fixtures/ngWM9fgd`, { headers }).then(r => r.json());
console.log(match.data.fixture.homeTeam.name); // "Leeds"

// Get events
const events = await fetch(`${BASE}/football/fixtures/GCxZ2uHc/events`, { headers }).then(r => r.json());
console.log(events.data.events[0].description); // "Loic Bade receives a red card!"
```

## Known Limitations

- Fixtures by league, teams list, team fixtures, and league stats endpoints return empty data during the API migration period. Use **fixture details by match ID** as the primary data source.
- Standings require `tournament_stage_id` (not the league slug). Find this ID via fixture details response (`tournament.tournament_stage_id`).
- League seasons returns limited data (0-1 seasons per league).
